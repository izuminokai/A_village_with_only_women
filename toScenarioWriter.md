# シナリオライターの方へ

このゲームのルールなどを流用してご自分のゲームを作ることは、特に禁止していません。ご自由に流用してください。  
以降は、流用する際に気をつけるべきポイントを記載します。

## ゲームデータ

- 閲覧のみ
    - [GM用データインデックス](data/index.html)
- データ取得
    - [GitHub](https://github.com/izuminokai/A_village_with_only_women)
    - Code -> Download ZIP でファイルがダウンロードできます

## ルールファイルは１万文字程度にする

アップロードするファイルが長すぎると、次のような不具合が起こります。

- ChatGPTが内容を一部無視する（存在しないと言い張る）
- ファイルアップロード直後に「ファイルが期限切れになりました」と言う

これらを防ぐには、アップロードする rules.txt の文字数を可能な限り短くし、ゲームシナリオやNPC設定などはWeb経由で参照させる必要があります。

とは言え、Web上のデータはアクセスするハードルが高いようで、なかなか参照してくれません。  
GMが常に意識すべき内容は rules.txt に残す必要があります。

## データを公開するサイトがAIに対応しているか確認

ゲームデータを公開するサイトはAIでも読み取れる場所でなくてはなりません。

- Notion: 不安定。読み取れる時もあるが、読み取れない時もある
- FC2: 不可。503エラー
- GitHub Pages: 可

ChatGPTは生のWebデータではなくクローラーを参照しているようで、公開してからChatGPTが認識できるようになるまで時間を要します（半日ぐらい？）

生成AIがアクセスできるサイトを使っていても尚、「アクセスできません」と言われる場合があります。  
これは推測ですが、どうやら生成AIが危険なサイトだと見なした場合にも「アクセスできません」と返すことがあるようです。  
PLが明示的に「アクセスしてください」と命じることで、この問題は解決できます（このことは rules.txt の[アクセス不可フォーマット]
にも記載しています）

## Web上のファイル形式はHTMLにする

txtやjsonは、生成AIには読みにくいようです。  
htmlにすると精度が上がります（恐らくクローラーの都合）

アップロードする rules.txt は、mdでも問題ないようです。

ちなみにこのゲームのデータは、GitHub Pagesで公開しています。  
GitHub Pagesには、mdファイルを自動でhtmlに変換する機能が備わっています。  
rules.txt の拡張子がmdではなくtxtなのは、このファイルをhtmlに変換せず、mdとしてダウンロードできるようにするためです。

## テーブルは使わない

htmlやmdは本来ならテーブルが使えますが、どうやらChatGPTは、テーブルのデータは上手く読めないようです。  
使わないほうが無難です。

## 流用できる範囲・必須項目

ルートフォルダ直下にあるファイルは、大部分がそのまま流用できるはずです。  
ただし、rules.txt の終盤には[ゲーム固有設定]などが入っていますから、そこは書き直す必要があります。

dataフォルダ内のシナリオやNPC設定などのファイルは、すべて書き直す必要があります。

- 以下3つのシナリオファイルは、導入時に使うため必須です。タイトル等は変えず、内容だけを書き換えてください。
    - [導入_キャラクター作成前](data/scenario/01_introBeforeCharacterCreation.html)
    - [導入_キャラクター作成後](data/scenario/02_introAfterCharacterCreation.html)
    - [導入後の流れ](data/scenario/03_afterTheIntro.html)
- [NPC設定]の以下の項目は、システム上必須です。
    - 初期好感度
    - 好感度の変化量
    - 好感度による行動変化
    - 浮気発覚時
    - 好きなタイプ
    - 嫌いなタイプ
    - 殺しやすさ
- [キャラクターシート](data/system/characterSheet.html)の以下の項目は、システム上必須です。
    - 最終目的
    - 能力値
        - 能力値の種類や合計値は、変更しても問題ありません
        - 範囲は0～3のままが無難です。4でギリギリ。5にすると破綻します（[成否判定]が失敗しなくなる）
            - 範囲を変える場合、rules.txt なども変更が必要です

# プロジェクト機能の可能性

ChatGPTのプロジェクト機能や、GeminiのNotebookLMを使えば、ルールだけでなくシナリオファイルなどもアップロードできるため、精度が高まるかもしれません。  
ただ、ChatGPTの無料版でアップロードできるのは最大5ファイルとなっているため、私はこの手段を採りませんでした。

[ChatGPT のプロジェクト](https://help.openai.com/ja-jp/articles/10169521-projects-in-chatgpt)より

    Pro ユーザー：最大40ファイル、最大100人の共同作業者 
    Plus / Go ユーザー：最大25ファイル、最大10人の共同作業者
    無料ユーザー：最大5ファイル、最大5人の共同作業者

PlusやGoでさえも、最大25ファイルです。  
このゲームは30ファイルを超えるため、仮にこれを採用するとしても、切り詰める必要があります。

また、プロジェクトでは、考える（思考）モードが使えないようなので、そこも不安要素です。
