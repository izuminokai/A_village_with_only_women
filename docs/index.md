# 生成AI TRPG「女性しかいない村」

## 推奨プレイ環境

- ChatGPTの無料版にてテスト実施
- **考える（Thinking）モードを強く推奨**

## 遊び方

1. [ゲームの開始](https://raw.githubusercontent.com/izuminokai/A_village_with_only_women/refs/heads/main/docs/gameStart.md)
   のテキストをすべてコピーし、生成AIチャットに貼り付けてください。
2. 生成AIの指示通りに、自分のキャラクターを作成してください。
3. <a href="rules.txt" download="ルール.md">ルール</a> をダウンロードし、生成AIへアップロードしてください。
4. 以降は、生成AIの指示に従ってください。

## シナリオライターの方へ

このゲームのルールなどを流用してご自分のゲームを作ることは、特に禁止していません。ご自由に流用してください。  
以降は、流用する際に気をつけるべきポイントを記載します。

### ルールファイルは１万文字程度にする

ChatGPTは、ファイルが長すぎると勝手に内容を読み飛ばして無視します。  
これを防ぐには、ファイルの文字数を10,000文字程度に収める必要があります。

### データを公開するサイトがAIに対応しているか確認

上述の都合上、ゲームシナリオやNPC設定などはファイルではなく、Web上に公開してそれを参照させる必要があるのですが、公開するサイトはAIでも読み取れる場所でなくてはなりません。

- Notion : 不安定。読み取れる時もあるが、読み取れない時もある
- FC2 : 不可。503エラー
- GitHub Pages : 可

ChatGPTは生のWebデータではなくクローラーを参照しているようで、公開してからChatGPTが認識できるようになるまで時間を要します（半日ぐらい？）

生成AIがアクセスできるサイトを使っていても尚、「アクセスできません」と言われる場合があります。  
これは推測ですが、どうやら生成AIが危険なサイトだと見なした場合にも「アクセスできません」と返すことがあるようです。  
PLが明示的に「アクセスしてください」と命じることで、この問題は解決できます（このことは「ルール」の[アクセス不可フォーマット]
にも記載しています）

### Web上のファイル形式はHTMLにする

txtやjsonは、生成AIには読みにくいようです。  
htmlにすると精度が上がります（恐らくクローラーの都合）

アップロードするルールファイルは、mdでも問題ないようです。

### テーブルは使わない

htmlやmdは本来ならテーブルが使えますが、どうやらChatGPTは、テーブルのデータは上手く読めないようです。  
使わないほうが無難です。
